# 贡献指南

感谢您对本项目的兴趣！我们欢迎各种形式的贡献，包括代码改进、文档完善、bug报告和功能建议。

## 如何贡献

### 1. 报告Bug

如果您发现了bug，请创建一个Issue并包含以下信息：

- 清晰的bug描述
- 复现步骤
- 预期行为
- 实际行为
- 环境信息（Python版本、操作系统等）

### 2. 提交功能建议

如果您有新的功能想法，请创建一个Issue并描述：

- 功能的目的和用途
- 预期的行为
- 可能的实现方式

### 3. 提交代码改进

我们欢迎Pull Request！请按照以下步骤操作：

1.  **Fork本仓库**
2.  **创建特性分支** (`git checkout -b feature/YourFeatureName`)
3.  **进行更改并提交** (`git commit -m 'Add YourFeatureName'`)
4.  **推送到分支** (`git push origin feature/YourFeatureName`)
5.  **打开Pull Request** 并清晰地描述您的更改

### 4. 改进文档

如果您发现文档有误或可以改进，请直接提交PR或创建Issue。

## 代码规范

- 遵循PEP 8风格指南
- 为新功能添加适当的注释和文档字符串
- 确保代码可读性和可维护性
- 在提交前进行本地测试

## 提交信息规范

请使用清晰、简洁的提交信息：

```
<type>: <subject>

<body>

<footer>
```

其中`<type>`可以是：

- `feat`: 新功能
- `fix`: bug修复
- `docs`: 文档改进
- `style`: 代码风格改进
- `refactor`: 代码重构
- `test`: 测试相关

## 许可证

通过提交代码，您同意您的贡献将在MIT许可证下发布。
