# API密钥说明

本项目使用了两种不同的API密钥，它们的作用和使用场景完全不同。请务必区分清楚。

---

## 1. `AGENT_API_KEY` (服务访问密钥)

**用途：** 用于**认证对本Agent API服务的访问**。

这是您作为API服务提供商，提供给您的客户的密钥。任何想要调用`/v1/extract`端点的客户端，都必须在HTTP请求的`X-API-Key`头中提供这个密钥。

- **谁来设置：** 您（API服务提供商）
- **如何使用：** 客户端在请求头中提供。

## 2. `llm_api_key` (客户自带的LLM密钥)

**用途：** 客户在调用API时，**选择性提供**，用于指定他们自己的LLM。

这是一个高级功能，允许客户使用他们自己的LLM账户（例如，他们自己的OpenAI、Azure、智谱等账户）来执行底层的智能抽取。如果客户不提供这个以及`llm_base_url`和`llm_model_name`，系统将使用默认的智谱GLM-4.5-Flash模型。

- **谁来设置：** 您的客户（在API请求中）
- **如何使用：** 在`POST /v1/extract`的请求体中作为`llm_api_key`字段提供。

---

## 总结

| API Key | 作用 | 提供者 |
| :--- | :--- | :--- |
| `AGENT_API_KEY` | **访问**您的Agent API | 您（服务商） |
| `llm_api_key` | **指定**Agent内部使用的LLM | 您的客户（可选） |

简单来说：
- 您的客户用`AGENT_API_KEY`来调用您的服务。
- 您的客户**可以选择**提供`llm_api_key`来覆盖默认的LLM，从而使用他们自己的LLM账户。
